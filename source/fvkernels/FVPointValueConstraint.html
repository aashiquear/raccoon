FVPointValueConstraint

This object implements the residuals that enforce the constraint

using a Lagrange multiplier approach. E.g. this object enforces the constraint that the value of in the element containing point match .

warningwarning

The contribution to the diagonal of the system of this kernel is null, which introduces a saddle point. Make sure to use a NONZERO shift in your preconditioner.

Example input syntax

In this example, the value of the variable v at 0.2 0 0 is set using a FVPointValueConstraint. In combination with a single Dirichlet boundary condition, this makes the numerical problem accept a single numerical solution, and be well-posed.

[Variables]
  [v]
    type = MooseVariableFVReal
  []
  [lambda]
    type = MooseVariableScalar
  []
[]

[FVKernels]
  [diff]
    type = FVDiffusion
    variable = v
    coeff = coeff
  []
  [average]
    type = FVPointValueConstraint
    variable = v
    phi0 = 13
    lambda = lambda
    point = '0.3 0 0'
  []
[]
(moose/test/tests/fvkernels/constraints/point_value.i)

Input Parameters

  • lambdaLagrange multiplier variable

    C++ Type:std::vector<VariableName>

    Controllable:No

    Description:Lagrange multiplier variable

  • phi0What we want the average value of the primal variable to be.

    C++ Type:double

    Controllable:No

    Description:What we want the average value of the primal variable to be.

  • pointThe XYZ coordinates of the points where the value shall be enforced.

    C++ Type:libMesh::Point

    Controllable:No

    Description:The XYZ coordinates of the points where the value shall be enforced.

  • variableThe name of the finite volume variable this kernel applies to

    C++ Type:NonlinearVariableName

    Controllable:No

    Description:The name of the finite volume variable this kernel applies to

    Description:The list of blocks (ids or names) that this object will be applied

    Default:1

    C++ Type:unsigned short

    Controllable:No

    Description:The number of layers of elements to ghost.

    Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    Default:False

    C++ Type:bool

    Controllable:No

    Description:Whether to use point neighbors, which introduces additional ghosting to that used for simple face neighbors.
