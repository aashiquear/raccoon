DerivativeMultiPhaseMaterial

buildconstruction:Undocumented Class

The DerivativeMultiPhaseMaterial has not been documented. The content listed below should be used as a starting point for documenting the class, which includes the typical automatic documentation associated with a MooseObject; however, what is contained is ultimately determined by what is necessary to make the documentation clear for users.

Two phase material that combines n phase materials using a switching function with and n non-conserved order parameters (to be used with SwitchingFunctionConstraint*).

Overview

Example Input File Syntax

  • fi_namesList of free energies for the n phases

    Controllable:No

    Description:List of free energies for the n phases

    Default:0

    Controllable:No

    Description:Energy barrier for the phase transformation from A to B

  • argsArguments of the fi free energies - use vector coupling

    C++ Type:std::vector<VariableName>

    Controllable:No

    Description:Arguments of the fi free energies - use vector coupling

    Description:The list of blocks (ids or names) that this object will be applied

    C++ Type:std::vector<BoundaryName>

    Description:The list of boundaries (ids or names) from the mesh where this boundary condition applies

    Default:True

    Controllable:No

    Description:When false, MOOSE will not call compute methods on this material. The user must call computeProperties() after retrieving the MaterialBase via MaterialBasePropertyInterface::getMaterialBase(). Non-computed MaterialBases are not sorted for dependencies.
